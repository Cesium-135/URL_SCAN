1. Basic Task Overview
Write a complete Python script that reads a list of URLs from an Excel file, simulates manual browser access to each URL, records the full redirection path of each jump, classifies each URL and jump step according to specified rules, and exports the results to a structured Excel file with two sheets (summary and detail). The script must support automatic redirection and manual click-triggered redirection, adapt to bilingual English/French pages, and include detailed bilingual comments.

2. Core Function Requirements
Read URLs from input Excel: url_list.xlsx, sheet named 'input_url', cell begin from A1
Standardize and validate URLs, auto-complement http/https, deduplicate.
Simulate real browser access, capture full jump chain: server 3xx redirect, JS/meta client redirect, page navigation.
Support simulating user clicks to trigger jumps (for buttons/links leading to Espace or login).
Capture jumps in new windows/tabs opened after click.
Implement breakpoint resume: skip processed URLs to avoid duplicate execution.
Support retry mechanism for failed access; single URL failure shall not interrupt the whole task.
Export results to Excel with two sheets: summary table and jump detail table.

3. URL Classification Rules (Strict)
Classify each jump step into 4 levels in the following order:
Public front – public entrance page
Espace – business space page (after front-end, before authentication)
Authentication – login/security module (login page, token, IAM, SSO)
Backend/API – back-end service, API interface/gateway
All classification and element matching only support English and French, no Chinese.

4. Click Simulation & Element Location Rules
Automatically locate clickable elements for Espace access and login in bilingual EN/FR.
Priority: attribute matching → text matching → role matching.
Support keywords:
Login: Login, Sign in, Connexion, Se connecter
Espace: Espace, My espace, Accéder à l'espace, Entrer
Simulate real user behavior: scroll into view, wait visible & enabled, normal click.
After click, wait for network idle and continue capturing the jump chain.
Merge the pre-click and post-click jump chains into one complete path.

5. Jump Scenarios to Support
Automatic redirection (direct jump after entering URL: 3xx, JS, meta).
Click-triggered jump (must click button/link to go to Espace/login).
Jumps within the same tab & jumps in new windows/tabs.
Internal systems with HTTPS/self-signed certificates (ignore SSL errors).
Delayed jumps & single-page application (SPA) front-end routing jumps.

6. Code & Output Requirements
Use Playwright for browser automation (stable for JS rendering and jumps).
Use openpyxl for Excel reading and writing.
All code functions with detailed Chinese-English bilingual comments.
Global configuration at the top for easy modification without changing core logic.
Output Excel includes:
Summary: raw URL, final URL, jump count, full path, main classification, click triggered or not, status.
Detail: each jump step, URL, type, trigger method (auto/click), classification, timestamp.
Friendly console log, execution statistics, clear error messages.

7. Constraints & Environment
English

Can be packaged into a standalone EXE using PyInstaller for distribution.
Playwright Chromium browser required; support proxy and headless mode.

#V2 Prompt
I have a Python script (script.py) that reads a list of URLs from an Excel file, simulates a real browser (Playwright) to visit each URL, captures the full redirection chain (including automatic redirects and clicks), classifies each URL into categories, and writes the results to an Excel file. The current version has several problems that need to be fixed. Below are the issues and the specific improvements I require.

Existing problems with the current script:
Hard‑coded file paths – The input/output Excel paths are hard‑coded, making debugging and deployment difficult.
→ Change to accept command‑line arguments (-i and -o) and prompt interactively if not provided. Use bilingual (English/Chinese) hints.
Insufficient bilingual output – Many comments, console messages, and Excel headers are only in Chinese.
→ Make all comments, logs, console output, and Excel headers strictly bilingual (English + Chinese).
Cookie consent popups not handled – Some pages get stuck because they require accepting cookies.
→ Add automatic cookie consent handling (EN/FR keywords, main frame + iframes, retry mechanism).
Pages requiring login/certificates – Some backend/admin pages ask for credentials or client certificates. The script cannot handle them, but it must detect and record this requirement, then close the browser cleanly.
→ Detect authentication/certificate prompts and set a flag auth_required = True. Ensure the browser always closes properly.
Poor error classification – Only a generic error is recorded. Need to extract the specific error code (e.g., ERR_CONNECTION_CLOSED, HTTP status codes) and store it in a separate column.
→ Extract error codes using regex, classify the error type (Connection Error, Timeout Error, Certificate Error, 4XX/5XX, Auth Required), and add columns for “Error Code” and “Error Sub‑Category”.
Coarse classification – Currently only 4 main categories: Public front, Espace, Authentication, Backend/API. I need finer sub‑categories, but I don’t know the exact criteria.
→ Keep the 4 main categories but add a sub_class field. Define sub‑categories based on URL keywords and page features (e.g., for Authentication: SSO/Auth Service, Login Page, Token/Credential). Provide configuration rules so sub‑categories can be adjusted later.
Cloud server deployment – The script will run on a cloud server (headless, no GUI).
→ Support headless mode, proxy configuration, ignore HTTPS errors, save logs and charts as files.
No final statistics or charts – After scanning, only the Excel file is produced. Need detailed statistics and visual charts.
→ Print statistics (total, success/fail, auth required, category distribution, error code distribution) and generate a PNG chart (pie chart for main categories + bar chart for error codes) with count labels and professional brand colours.
Single‑threaded, slow – With current timeout, it processes about one URL every 10 seconds.
→ Optimize with multi‑threading (configurable concurrency, default 10–30 threads) while keeping resource cleanup safe.

Additional improvements (derived from later discussions):
Breakpoint resume – If the output Excel already exists, skip already processed URLs to avoid re‑scanning.
→ Read the “Status” column of the summary sheet; only process URLs that are not already marked “Success”.
Click simulation enhancements – For sites that require a button click to reach Espace or login pages, automatically locate and click the element (EN/FR keywords, priority selectors). Capture jumps that open in new windows/tabs and merge them into the full chain.
Error robustness – Avoid Page.content: Target page, context or browser has been closed errors. Use safe cleanup and avoid fetching intermediate page HTML that might close the context.
Chart styling – Apply the official BNP Paribas brand colours (green primary palette) to the pie and bar charts. Show both count and percentage on the pie chart, and count labels on top of each bar in the bar chart.
Configuration flexibility – All parameters (timeout, retries, headless, proxy, click keywords, cookie keywords, classification rules, Excel column letter, etc.) must be grouped at the top of the script in a CONFIG dictionary. No hard‑coded values inside functions.
Testing – Provide a separate test script (or instructions) that can generate a simulated set of scan results to verify the chart generation without actually running the browser.

Output Excel: Two sheets – summary (one row per URL with all metadata) and jump_chain (one row per redirection step). Bilingual headers.
